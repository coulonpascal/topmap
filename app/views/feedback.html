The node, as it exists, is a limited test node and is subject to a development cycle:

  1. Secure time from GI users
  2. User considers different scenarios for using the node, then tests the scenarios practically through the available services and the interface.
  3. Discussion around findings, between user and node team through an interview, covering:
    1. what the user found to be the node’s strengths;
    2. what the user found to be the node’s weaknesses;
    3. ideas for improvement of the node, what developments could help to save users time; and
    4. thoughts on priorities for all discussed changes
  5. The required changes would then be prioritised and a limited number would be scheduled for development.

Any comments on the node for consideration within the first development cycle need to be made by Friday 26 February, as the first development cycle is scheduled to complete 11-18 March.

Please get in touch with us using the email address below if you can work through the node and contribute ideas on improving it, for your individual and organisational use.
