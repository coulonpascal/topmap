About the node

This portal has been created to share earth observation data and other widely used geographic information (GI) datasets across the DEFRA Group. Delivering centrally managed datasets is expected to save time and money as group members will not have to manage these large datasets in-house. The node will also allow for cloud-hosted data processing facilities which should enable larger scale data processing which again does not have to be hosted and managed through local infrastructure; the outputs of which can be readily shared where appropriate.

The portal is in the very early stages of development and we need your help to make it fit for your purpose. As you use this portal, please send us your comments on how to improve the service it provides, and make it fit for your purposes. We welcome comments on any aspect of the system: functionality, content, interface and layout, and we will feed these into the development and release cycle (see Roadmap).

Users should be aware that this site will undergo rapid development of both content and functionality is in progress. There will also be changes to the interface. Major planned and delivered changes will be tracked through the Roadmap.

User should also be aware that there are a number of EO data portals, both live and in development, which are acquiring similar EO datasets. Where the cost of managing a dataset becomes greater than an external service delivering the same product, this service will no longer supply the data. Therefore this node will be influenced by competing services and will only continue as long as there is justification for the investment.

Using the node

At present, the data contained within this node may only be viewed through this site, using the mapper facilities. It will very soon be able to be downloaded from this site; the download function is scheduled for imminent release.

The data contained within this site can also be used within GI software, through web feature services (WFS) or web mapping services (WMS).

QGIS

  1. Select one of the following:
    1. Add WFS Layer from a Server
    2. Add WCS Layer from a Server
    3. Add WMS Layer from a Server
  3. Give the connection a title e.g. 'Pilot collaboration node'
  4. Enter the URL:
    1. For WFS - {{wfs_capabilities_url}}
    2. For WCS - {{wcs_capabilities_url}}
    3. For WMS - {{wms_capabilities_url}}
  6. Click 'Connect' (you don't need username or password)
  7. Select the dataset of interest, then click 'Add'

ArcMap

  1. Select: 'File' – 'Add Data' – 'Add Data', then one of the following:
    1. Add WCS Layer from a Server
    2. Add WMS Layer from a Server
    3. Add WMTS Layer from a Server
  3. Enter the following URL:
    1. For WCS - {{wcs_capabilities_url}}
    2. For WMS - {{wms_capabilities_url}}
    3. For WMTS - {{wmts_capabilities_url}}
  5. Click 'Connect' (you don't need username or password)
  6. Drill down through the new connection link using 'Add'

It is intended that users will soon be able to set-up processing within the node, rather than downloading the data for local processing. There are also plans for large scale data processing through node to node transfer, for example for processing all of the available satellite data on a regular schedule.

Background

The node is a product of the Earth Observation Centre of Excellence (EOCoE); DEFRA and the UK Space Agency jointly agreed a 'road map' for realising benefits from earth observation data. The EOCoE looks to maximise on potential benefits from applying earth observation data, as received from the European Space Agency/European Commission's Copernicus initiative, and is running 11 investigations to this end. This node will capture EO data from various sources: ESA satellites, satellites hosted by individual member states, UK in-situ sensors.

This node will also help to share key GI datasets across DEFRA; in line with DEFRA's GI strategy. The DEFRA group all require access to satellite data, cartographic data, and other GI datasets, and at present will largely manage individual copies of each dataset locally. There is a significant overlap in the specific datasets managed by each member of the group; this node will enable central management and deployment of key datasets, reducing management costs and improving consistency (dataset versioning).
